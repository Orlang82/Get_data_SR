# TASKS_chart_7s_new.md

## Задача: chart_7s (обновлённая спецификация)

**Цель:** Создать скрипт построения графика «Динаміка мінімального розміру ринкового ризику» на основе данных из Excel-таблицы `tDB_History_2` и вставки готового изображения в рабочую книгу. График отображает динамику четырёх рядов данных (столбчатая гистограмма + три линии) за последние 12 наблюдений.

---

## Входные данные

- Таблица `tDB_History_2` на листе `sys` рабочей книги Excel (читается через xlwings):
  - `Дата` — дата наблюдения (datetime)
  - `МРРР` — значение МРРР в гривнях (ряд: столбчатая гистограмма)
  - `Валютний ризик` — значение валютного риска в гривнях (ряд: линия)
  - `Процентний ризик` — значение процентного риска в гривнях (ряд: линия)
  - `Товарний ризик` — значение товарного риска в гривнях (ряд: линия, вторичная ось)

---

## Первоначальные действия

1. Прочитать таблицу `tDB_History_2` с листа `sys` через xlwings API: `sheet.api.ListObjects("tDB_History_2")`
2. Преобразовать диапазон таблицы в `pandas.DataFrame` (заголовок — первая строка)
3. Убедиться, что колонка `Дата` имеет тип `datetime`, числовые колонки — `float`
4. Отобрать **последние 12 строк** по порядку (по колонке `Дата` или по позиции — строки уже отсортированы по дате)
5. Вычислить значения в тыс. грн. (делить каждое значение на 1 000; округлять до целых) — только для **подписей данных**, для построения графика использовать исходные значения в грн.

## ВАЖНАЯ ИНФОРМАЦИЯ

### 1. Обработка NaN в ряде `Процентний ризик`

В ряде данных `Процентний ризик` значения присутствуют **только за последние 2 периода**, все предыдущие записи содержат `NaN`. Критические требования:

- **NaN должен игнорироваться полностью** — matplotlib автоматически пропускает `NaN` при построении линий
- **График строится только по фактическим данным** — если доступно только 2 значения, линия строится только между этими двумя точками (с учётом их дат по оси X)
- **Не пытаться интерполировать или заполнять NaN** — это исказит визуальное восприятие динамики
- **Подпись данных выводится только для существующих значений** — для `NaN` подписи не отображаются

### 2. Ограничение значений в режиме разрыва

К ряду `Процентний ризик` применяется следующее правило:

- Если активирован режим разрыва (один из столбцов `МРРР` превышает `MRRR_BREAK_MULTIPLIER × медиана`), то:
  - **Линии `Процентний ризик` ограничиваются порогом `y_cap`** для корректного масштабирования основной оси Y
  - Это предотвращает «сжатие» линий к оси X при наличии аномально высокого столбца МРРР
  - значения подписей данных `Процентний ризик` сохраняются оригинальными

### 3. Последовательность отрисовки

Для корректного наложения элементов соблюдайте порядок:
1. Столбцы гистограммы (МРРР)
2. Символы разрыва (если есть)
3. Линии на основной оси (Валютний, Процентний)
4. Линия на вторичной оси (Товарний)
5. Подписи данных (над всеми элементами)
6. Легенда и заголовок

## Логика обработки

### Определение «разрыва» на столбцах гистограммы (МРРР)

Задача — не допустить, чтобы один аномально высокий столбец «прижал» остальные столбцы и линии к оси X.

1. Вычислить медиану значений `МРРР` по выбранным 12 точкам.
2. Если **хотя бы один** столбец превышает `МРРР_BREAK_MULTIPLIER × медиана` (параметр конфигурации, значение: `3`) — активировать режим разрыва.
3. В режиме разрыва:
   - Ограничить визуальную высоту оси Y значением `y_cap`, которое вычисляется автоматически: `y_cap = медиана × MRRR_CAP_MULTIPLIER` (параметр конфигурации, значение: `2.5`)
   - Для столбцов, чьё значение превышает `y_cap`, отрисовать столбец обрезанным по высоте `y_cap`
   - **Пропорциональная обрезка** (`BREAK_PROPORTIONAL = True`):
     - Каждый обрезанный столбец имеет индивидуальную высоту: `bar_heights[i] = y_cap + (mrrr[i] - y_cap) * k`
     - Коэффициент сжатия `k` подбирается так, чтобы максимальный столбец не превышал `y_cap * 1.3`
     - Это сохраняет относительные пропорции: столбец 28 будет выше столбца 20
   - Поверх каждого обрезанного столбца нарисовать символ разрыва: две короткие косые полосы (∿∿), реализованные через `ax.plot()` с зигзагообразными координатами
   - Подпись данных для обрезанных столбцов выводить **над символом разрыва**
4. Для столбцов без разрыва подпись данных выводить **над столбцом** стандартным образом.

### Форматирование подписей данных

- Значения для подписей = исходное значение / 1000, округлить до целых
- Формат: `"{:,.0f}".replace(",", " ")` — разделитель тысяч: пробел (украинский формат), без десятичных знаков
- Подписи выводить для **всех** рядов данных (МРРР, Валютний ризик, Процентний ризик, Товарний ризик)

### Положение подписей

Для каждого ряда данных задаётся индивидуальное положение подписи:
- `LABEL_POSITION_VAL = 'top'` — подпись над точкой (Валютний ризик)
- `LABEL_POSITION_PCT = 'bottom'` — подпись под точкой (Процентний ризик)
- `LABEL_POSITION_TOVAR = 'top'` — подпись над точкой (Товарний ризик)

### Цвета подписей

Для контраста с фоном и линиями задаются индивидуальные цвета:
- `LABEL_COLOR_VAL = '#FFFFFF'` — белый для Валютний ризик (зелёная линия)
- `LABEL_COLOR_PCT = '#1F3864'` — тёмно-синий для Процентний ризик (жёлтая линия)
- `LABEL_COLOR_TOVAR = '#FFFFFF'` — белый для Товарний ризик (оранжевая линия)

## Структура скрипта

Рекомендуемое разделение на функции:

1. **`_setup_logger() → logging.Logger`** — настраивает логгер с записью в файл `logs/chart_7s.log` или возвращает disabled-логгер
2. **`get_chart_data() → pd.DataFrame`** — читает таблицу `tDB_History_2`, возвращает последние `N_LAST` строк
3. **`format_label(value_grn) → str`** — форматирует значение в грн. → подпись в тыс. грн. с пробелом-разделителем
4. **`draw_break_symbol(ax, x, y_cap)`** — рисует символ разрыва (два зигзага) на указанном столбце через `ax.plot()`
5. **`build_chart(df) → matplotlib.figure.Figure`** — строит полную фигуру по данным; вся логика разрыва здесь:
   - Вычисление медианы и порога разрыва
   - Пропорциональная обрезка столбцов (если `BREAK_PROPORTIONAL = True`)
   - Отрисовка столбцов и символа разрыва
   - Отрисовка линий на основной и вторичной осях
   - Подписи данных с индивидуальным положением и цветом
   - Настройка осей, заголовка, легенды
6. **`save_chart(fig) → str`** — сохраняет фигуру во временный файл, возвращает путь
7. **`insert_chart_to_excel(img_path)`** — удаляет старое изображение по `IMAGE_NAME`, вставляет новое в ячейку `IMAGE_CELL`
8. **`create_market_risk_chart()`** — **основная публичная функция**, последовательно вызывает 2→5→6→7, оборачивает в try/except с print-логированием

## Ключевые особенности реализации

1. **Пропорциональная обрезка столбцов** — сохраняет визуальное соотношение между высокими столбцами, что важно для восприятия масштаба выбросов
2. **Индивидуальное положение подписей** — для каждого ряда данных задаётся своё положение (`top`/`bottom`)
3. **Индивидуальные цвета подписей** — подбираются для контраста с цветом линии (белый на зелёном/оранжевом, тёмно-синий на жёлтом)
4. **Два уровня отступов подписей** — общий `LABEL_OFFSET_Y` и индивидуальные `LABEL_OFFSET_VAL/PCT/TOVAR`
5. **Ограничение линий в режиме разрыва** — линии `Валютний` и `Процентний` тоже ограничиваются порогом `y_cap` для масштабирования оси
6. **Отладочный вывод диапазонов** — при чтении данных выводится min/max для каждого ряда
